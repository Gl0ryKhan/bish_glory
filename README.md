# 🏠 Bish Glory — Предсказание цен на квартиры в Бишкеке

> ML-проект по оценке рыночной стоимости квартир в Бишкеке на основе данных с сайта house.kg.  
> Реализован полный ML-пайплайн: парсинг → EDA → feature engineering → линейная модель → CatBoost → Gradio-интерфейс.

---

## 🎯 Цель проекта

Предсказать цену квартиры в **USD** по её характеристикам с точностью medianAPE ≈ **9.5%**.

---

## 📊 Данные

| Параметр | Значение |
|---|---|
| Источник | house.kg (парсинг) |
| Файл | house_kg_sell_flats_1783230474.csv |
| Объём | ~12 000 объявлений о продаже квартир |
| Целевая переменная | usd_price |

### Признаки

| Признак | Описание |
|---|---|
| lat, lon | GPS-координаты |
| Серия | Серия дома (элитка, хрущёвка, 104 серия...) |
| Отопление | Тип отопления |
| Состояние | Состояние (евроремонт, ПСО...) |
| rooms | Кол-во комнат |
| square | Площадь м² |
| house_type | Тип дома (монолит, кирпич, панель) |
| build_year | Год постройки |
| floor / total_floors | Этаж / этажность |
| district | Район города |
| Документы | ДДУ, техпаспорт, красная книга, ДКП |

---

## 🏗️ Архитектура ML-пайплайна

```
Сырые данные (house.kg CSV)
        |
        v
  [EDA & Анализ]        <- boosting.ipynb, logistic_reg.ipynb
        |
        v
  [Feature Engineering]
  * log1p(usd_price)     <- таргет
  * floor_ratio, is_first/last_floor
  * build_year_is_missing
  * OHE: Тип предложения
  * Категориальные: Серия, Состояние, house_type, district
        |
        v
  [Модели]
  +-- Logistic Regression (бейзлайн)
  +-- CatBoost (основная)
  +-- Optuna (гиперпараметры)  -> R2 = 0.9044
        |
        v
  [Финальная модель]
  final_catboost_model.cbm
        |
        v
  [Gradio Web App]       <- app.py
```

## 🤖 Модели

### 1. Бейзлайн — Linear Regression
- R² ≈ 0.74 (Train)
- Ноутбук: logistic_reg.ipynb

### 2. CatBoost (финальная модель)

| Метрика | Значение |
|---|---:|
| **R² (log-шкала)** | **0.8908** |
| RMSE (log) | 0.169 |
| R² (USD) | 0.8414 |
| MAE (USD) | 16 070 |
| MAPE | 12.38% |
| **median APE** | **9.49%** |

Файл модели: `final_catboost_model.cbm` (4.7 MB)

### 3. Optuna — подбор гиперпараметров

Лучшая конфигурация (35 попыток):

| Параметр | Значение |
|---|---|
| iterations | 1309 |
| learning_rate | 0.1152 |
| depth | 7 |
| l2_leaf_reg | 0.0475 |
| bagging_temperature | 0.7866 |
| random_strength | 0.0247 |
| **R²** | **0.9044** |

## 🔑 Ключевые решения

1. **Таргет log1p(usd_price)** — логарифм убирает скос распределения цен (skew=3.36)
2. **Категориальные признаки нативно** — CatBoost обрабатывает Серия/Состояние/Район без OHE
3. **Сырые координаты (lat, lon)** — деревья ловят нелинейную геозависимость
4. **floor_ratio** — относительный этаж вместо абсолютного
5. **build_year_is_missing** — флаг пропуска как признак
6. **Optuna** — байесовский поиск гиперпараметров (35 итераций, R²=0.9044)
